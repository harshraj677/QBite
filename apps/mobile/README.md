# QBite Mobile

Flutter customer app. See the repo root [`README.md`](../../README.md) and [`docs/ARCHITECTURE.md`](../../docs/ARCHITECTURE.md#2-mobile-architecture-flutter) for full context — this file only covers what's specific to running this app.

## Structure

- `lib/core/` — design tokens (`theme/`), logging, error handling, constants, responsive/context utilities. No feature logic.
- `lib/config/` — environment config, Riverpod core providers (DI), GoRouter route table.
- `lib/shared_widgets/` — reusable UI components (buttons, inputs, cards, loading/empty/error states, animations).
- `lib/features/<name>/{data,domain,presentation}/` — feature-first, Clean Architecture layering. Empty until each feature is implemented.
- `lib/l10n/` — ARB source files; `lib/l10n/generated/` is build output (gitignored, regenerated by `flutter gen-l10n` / `flutter pub get`).

## Flavors

Three build flavors, each with its own entrypoint and `.env.<flavor>` file (copy from `.env.example`, never commit the real files):

```
flutter run -t lib/main_development.dart   # default — same as plain `flutter run`
flutter run -t lib/main_staging.dart
flutter run -t lib/main_production.dart
```

## Common commands

```bash
flutter pub get       # install dependencies + regenerate localization
flutter analyze        # static analysis — must return zero issues
flutter test             # widget tests
dart format lib test      # formatting
```
